    ## Voraussetzungen (D-09)
    - Docker Desktop 4.x (Windows/macOS) oder Docker Engine 24+ (Linux)
    - Git (zum Klonen des Repos)
    - Mindestens 8 GB RAM (ML-Modelle: bge-m3 ~2 GB, bge-reranker-v2-m3 ~2 GB)
    - Mindestens 10 GB freier Festplattenspeicher (Docker Images + ML-Modelle ~4 GB + Qdrant-Daten)
    - Windows 10/11, macOS 12+, oder Linux mit Docker-Unterstuetzung

    ## Schnellstart (Docker)
    Step-by-step with numbered steps:
    1. Repository klonen: `git clone https://github.com/your-username/paragraf.git && cd paragraf`
    2. Environment konfigurieren: `cp .env.example .env` (optional, Standardwerte funktionieren)
    3. Starten: `docker compose up --build`
    4. Oeffnen: `http://localhost:3847` (Frontend) oder `http://localhost:8000/api/health` (Backend direkt)
    5. Gesetze indexieren: Ueber die Web-Oberflaeche oder `curl -X POST http://localhost:3847/api/index -H "Content-Type: application/json" -d '{"gesetzbuch": "SGB IX"}'`

    ## Ports und Services
    Table matching CLAUDE.md architecture:
    | Service | Port | URL | Funktion |
    |---------|------|-----|----------|
    | Frontend (nginx) | 3847 | http://localhost:3847 | Web-App + API-Proxy |
    | Backend (FastAPI) | 8000 | http://localhost:8000 | REST-API direkt |
    | MCP-Server | 8001 | http://localhost:8001/mcp | MCP fuer Claude/LLM |
    | Qdrant | 6333 | http://localhost:6333 | Vektordatenbank |

    ## GPU-Setup (NVIDIA) (D-10)
    - Voraussetzung: NVIDIA-Treiber + NVIDIA Container Toolkit installiert
    - Starten mit GPU: `docker compose -f docker-compose.yml -f docker-compose.gpu.yml up --build`
    - Erklaerung: Setzt EMBEDDING_DEVICE=cuda, reserviert GPU fuer den Backend-Container
    - Pruefen ob GPU erkannt: `curl http://localhost:3847/api/settings/gpu`

    ## Konfiguration
    - Verweis auf .env.example fuer alle Environment-Variablen
    - Wichtigste Variablen erklaeren: QDRANT_HOST, EMBEDDING_MODEL, RERANKER_MODEL, LOG_LEVEL
    - Hinweis: Settings-Page in der Web-App ist read-only (Konfiguration nur ueber docker-compose.yml / .env)

    ## Lokale Entwicklung (D-11)
    ### Backend
    ```bash
    cd backend
    pip install -e ".[dev]"
    python -m paragraf --mode api --port 8000
    ```
    Voraussetzung: Python 3.12+, laufende Qdrant-Instanz

    ### Frontend
    ```bash
    cd frontend
    npm install
    npm run dev
    ```
    Voraussetzung: Node.js 22+, Vite-Dev-Server proxied /api nach localhost:8000

    ### Tests
    ```bash
    docker compose exec backend python -m pytest tests/ -v
    ```

    ## Troubleshooting (D-12)
    Haeufige Probleme mit Loesungen:
    - **Port belegt (3847, 8000, 6333):** `netstat -ano | findstr :8000` (Windows) / `lsof -i :8000` (Linux/Mac), Prozess beenden oder Port in docker-compose.yml aendern
    - **Docker nicht gestartet:** Docker Desktop muss laufen bevor `docker compose up` funktioniert
    - **Nicht genug RAM:** ML-Modelle brauchen ~4 GB RAM im Container. Docker Desktop Standardlimit erhoehen (Settings > Resources > Memory)
    - **Langsamer erster Start:** Normal — beim ersten Start werden ML-Modelle (~4 GB) heruntergeladen. Folgestarts sind schnell.
    - **GPU wird nicht erkannt:** NVIDIA Container Toolkit pruefen: `nvidia-smi` muss im Container laufen. Siehe GPU-Setup Abschnitt.
    - **Windows MAX_PATH:** Langes-Pfad-Problem vermeiden — Repository nicht in tief verschachtelte Ordner klonen.

    Use `http://localhost:3847` as primary URL in curl examples (Docker/nginx path). Note `http://localhost:8000` for local dev at the top.

    Structure (per D-14, grouped by function):

    ## Header
    Title: `# Paragraf REST-API`
    Intro paragraph: Kurze Erklaerung dass alle Endpoints unter `/api/` erreichbar sind. Zwei Zugriffswege:
    - Docker: `http://localhost:3847/api/...` (ueber nginx)
    - Lokal: `http://localhost:8000/api/...` (direkt)

    Note: Alle Anfragen und Antworten in JSON. Content-Type: application/json.

    ## Suche (4 endpoints)
    For each endpoint document:
    - HTTP method + path
    - 1-2 sentence German description of WHEN to use this (not just what it does)
    - Request body fields (name, type, required/optional, description)
    - curl example with realistic German legal query (e.g., "Kuendigungsschutz bei Schwerbehinderung")
    - Brief response description (key fields)

    Endpoints:
    1. `POST /api/search` — Hauptsuche. Hybrid aus Dense + Sparse Vektoren mit Reranking. Standard fuer die meisten Suchanfragen.
       Request: `{"anfrage": "...", "top_k": 5, "gesetzbuecher": ["SGB IX"], "reranking": true}`
    2. `POST /api/search/batch` — Mehrere Suchanfragen gleichzeitig. Sinnvoll wenn man verschiedene Aspekte einer Rechtsfrage gleichzeitig recherchiert.
    3. `POST /api/search/grouped` — Suchergebnisse nach Gesetz gruppiert. Zeigt welche Gesetze zu einer Frage relevant sind.
    4. `POST /api/search/multi-hop` — Mehrstufige Suche die Querverweisen folgt. Fuer komplexe Rechtsfragen die mehrere Gesetze betreffen.

    ## Empfehlungen (3 endpoints)
    5. `POST /api/recommend` — Aehnliche Paragraphen finden. Ausgehend von einem bekannten Paragraphen verwandte Vorschriften entdecken.
    6. `POST /api/recommend/grouped` — Aehnliche Paragraphen nach Gesetz gruppiert.
    7. `POST /api/discover` — Thematische Exploration. Ausgehend von Beispiel-Paragraphen oder Stichworten neue relevante Vorschriften entdecken.

    ## Nachschlagen (2 endpoints)
    8. `POST /api/lookup` — Konkreten Paragraphen nachschlagen. Wenn man genau weiss welchen Paragraphen man braucht.
    9. `POST /api/compare` — Zwei Paragraphen vergleichen. Zeigt beide Texte nebeneinander.

    ## Querverweise (2 endpoints)
    10. `POST /api/references/extract` — Referenzen aus Text extrahieren. Erkennt Verweise wie "nach § 2 SGB IX" in Freitext.
    11. `GET /api/references/{gesetz}/{paragraph}` — Verweisnetzwerk fuer einen Paragraphen. Zeigt wer auf diesen Paragraphen verweist und worauf er verweist.

    ## Gesetze (3 endpoints)
    12. `GET /api/laws` — Alle verfuegbaren Gesetze auflisten mit Indexierungsstatus. Optional: `?rechtsgebiet=Sozialrecht` filtern.
    13. `GET /api/laws/{gesetz}/structure` — Gliederung eines Gesetzes (Teile, Kapitel, Abschnitte).
    14. `GET /api/laws/{gesetz}/paragraphs` — Alle Paragraphen eines Gesetzes.

    ## Beratung (1 endpoint)
    15. `POST /api/counseling` — EUTB-Beratungsstellen suchen. Fuer Nutzer die professionelle Beratung brauchen.

    ## Index-Verwaltung (4 endpoints)
    16. `GET /api/index/status` — Indexierungsstatus pro Gesetz.
    17. `POST /api/index` — Gesetz indexieren (SSE-Stream mit Fortschritt). Body: `{"gesetzbuch": "SGB IX"}`.
    18. `POST /api/index/eutb` — EUTB-Beratungsstellen-Daten indexieren.
    19. `POST /api/indexes/ensure` — Qdrant-Collections anlegen falls nicht vorhanden.

    ## Snapshots (4 endpoints)
    20. `POST /api/snapshots` — Snapshot erstellen (Backup der Qdrant-Daten).
    21. `GET /api/snapshots` — Alle Snapshots auflisten.
    22. `POST /api/snapshots/{name}/restore` — Snapshot wiederherstellen.
    23. `DELETE /api/snapshots/{name}` — Snapshot loeschen.

    ## System (7 endpoints)
    24. `GET /api/health` — Health-Check. Prueft Backend, Qdrant-Verbindung, ML-Modelle.
    25. `GET /api/settings` — Aktuelle Konfiguration (read-only).
    26. `GET /api/settings/gpu` — GPU-Erkennungsstatus.
    27. `POST /api/models/download` — ML-Modelle herunterladen (SSE-Stream mit Fortschritt).
    28. `GET /api/models/status` — Download-/Ladestatus der Modelle.
    29. `GET /api/models/cache` — Cache-Groesse und -Pfad.
    30. (Bonus, discovered) `DELETE /api/models/cache` — Model-Cache loeschen.

    For EACH endpoint, include a curl example. Use `jq .` for readable output. Use realistic German legal queries as examples (per project convention: "SGB IX" not "SGB 9").

    Example curl pattern:
    ```bash
    curl -s http://localhost:3847/api/search \
      -H "Content-Type: application/json" \
      -d '{"anfrage": "Kuendigungsschutz bei Schwerbehinderung", "top_k": 5}' | jq .
    ```

    RDG-Disclaimer at the bottom (per project convention): Paragraf ist kein Rechtsberatungsdienst im Sinne des RDG. Die Ergebnisse sind allgemeine Informationen, keine individuelle Rechtsberatung.
