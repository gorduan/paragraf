## Interaction States

### ModelDownloadStep (Wizard Step)

| State | Visual | Behavior |
|-------|--------|----------|
| **Pre-download** | Disk space check result shown; two model cards with "Bereit zum Download" status | "Download starten" primary CTA enabled |
| **Disk space warning** | Warning banner (warning-50 bg, warning-600 border): "{X} GB frei, {Y} GB benoetigt" with "Pfad aendern" and "Trotzdem versuchen" buttons | Blocks download until user chooses |
| **Downloading (per model)** | Active model: indigo progress bar filling, metadata row below: "{downloaded} / {total} GB -- {speed} MB/s -- ca. {minutes} Min." Inactive model: gray "Wartet..." | "Weiter" disabled; "Abbrechen" link available |
| **Download error** | Error banner (error-50 bg): "Download fehlgeschlagen: {message}" with "Erneut versuchen" button. Retry count shown: "Versuch {n}/3" | Auto-retry 3x with backoff, then manual retry button |
| **Download complete (per model)** | Green checkmark icon + "Heruntergeladen" + size in GB | When all models complete: "Weiter" button enables |
| **All complete** | Both models show green checkmarks; success message: "Alle Modelle heruntergeladen" | "Weiter" button enabled, auto-advances after 1s |

### GpuDetectionStep (Wizard Step)

| State | Visual | Behavior |
|-------|--------|----------|
| **Detecting** | Spinner + "GPU wird erkannt..." | Auto-triggers on step entry |
| **GPU found** | GPU card: icon + GPU name + VRAM. Recommendation badge: "Empfohlen". Two buttons: "GPU aktivieren (Empfohlen)" primary, "CPU beibehalten" secondary | User chooses, then "Weiter" |
| **No GPU** | Info message (neutral bg): "Keine NVIDIA-GPU erkannt. Paragraf laeuft im CPU-Modus." | "Weiter" button immediately enabled; no action needed |
| **GPU activation failed** | Warning message: "GPU-Modus konnte nicht aktiviert werden. CPU-Modus wird verwendet." | Falls back to CPU silently; "Weiter" enabled |

### GPU Toggle (SettingsPage)

| State | Visual | Behavior |
|-------|--------|----------|
| **CPU active** | Toggle switch off; label "CPU-Modus aktiv" | Toggle clickable |
| **GPU active** | Toggle switch on (primary-600); label "GPU-Modus aktiv ({GPU name})" | Toggle clickable |
| **Switching** | Toggle disabled; spinner; "Backend wird neu gestartet..." text; HealthOverlay takes over | ~30s wait; auto-resolves when health returns |
| **Switch failed** | Toast/inline warning: "GPU-Modus konnte nicht aktiviert werden. CPU-Modus wird verwendet." Toggle reverts to CPU. | Automatic fallback per D-09 |
| **No GPU available** | Toggle disabled; tooltip: "Keine NVIDIA-GPU erkannt" | Cannot interact |
| **Web-only mode** | Toggle replaced by instructions text: "GPU-Wechsel erfordert Docker-Neustart. Fuehren Sie aus: docker compose -f ... up" | Read-only |

### Cache Management (SettingsPage)

| State | Visual | Behavior |
|-------|--------|----------|
| **Loaded** | Cache path (monospace), total size in GB, per-model table (name, size, status dot green/gray) | All read-only display |
| **Clearing** | "Cache loeschen" button shows spinner, disabled | Confirmation dialog appears first |
| **Confirmation dialog** | Modal overlay: "Modelle werden geloescht und beim naechsten Start automatisch neu heruntergeladen. Fortfahren?" with "Abbrechen" (secondary) and "Cache loeschen" (destructive/red) | Two-button confirmation per D-07 |
| **Cleared** | Success message: "Model-Cache geloescht." Per-model table shows "Nicht heruntergeladen" in gray | Next app start triggers download step |
| **Path change** | "Aendern" button opens OS folder dialog via Electron IPC | Shows new path after selection |
| **Error** | Inline error: "Cache konnte nicht geloescht werden: {reason}" | Retry available |

|  | [Icon] Embedding-Modell (BAAI/bge-m3)       2.2GB | |
|  | [============================-----]  78%           | |
|  | 1.7 / 2.2 GB -- 45 MB/s -- ca. 2 Min.             | |
|  +---------------------------------------------------+ |
|                                                        |
|  +---------------------------------------------------+ |
|  | [Icon] Reranker-Modell (BAAI/bge-reranker)  1.1GB | |
|  | [                                  ] Wartet...     | |
|  +---------------------------------------------------+ |
|                                                        |

- Model cards: `rounded-xl border border-neutral-200 dark:border-neutral-700 bg-white dark:bg-neutral-800 p-4`

- Metadata text: 12px caption, `text-neutral-500`
- Gap between model cards: 16px
- Container max width: `max-w-md` (matching StorageStep)

## Accessibility Requirements

| Requirement | Implementation |
|-------------|----------------|
| Progress bar ARIA | `role="progressbar"` with `aria-valuenow`, `aria-valuemin`, `aria-valuemax`, `aria-label="Embedding-Modell: 78%"` |
| Live progress updates | `aria-live="polite"` on progress metadata text; update throttled to every 2s to avoid screen reader spam |
| Toggle switch | `role="switch"` with `aria-checked`, `aria-label="GPU-Modus"` (matching existing dark mode toggle pattern) |
| Confirmation dialog | `role="alertdialog"` with `aria-labelledby` and `aria-describedby`; focus trapped inside dialog; Escape closes |
| Error announcements | `role="alert"` on download error messages (matching existing pattern) |
| Stepper navigation | `aria-current="step"` on active step (already in SetupWizard) |
| Keyboard | Tab order follows visual order; Enter/Space activates buttons; Escape cancels dialog |
| Reduced motion | Progress bar transition respects `prefers-reduced-motion` (existing CSS handles this) |
