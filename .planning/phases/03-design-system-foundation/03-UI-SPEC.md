# Phase 3 — UI Design Contract

> Visual and interaction contract for the Design System Foundation phase. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (manual Tailwind + Radix) |
| Preset | not applicable |
| Component library | Radix UI Primitives (headless) |
| Icon library | lucide-react ^0.460.0 (existing) |
| Font | System font stack (existing default) |

**Radix packages to install (D-06):**
- `@radix-ui/react-dialog` -- Modal/dialog with focus trap, Escape close, ARIA
- `@radix-ui/react-select` -- Accessible select/dropdown with keyboard nav
- `@radix-ui/react-tooltip` -- Delayed tooltip with accessible labeling
- `@radix-ui/react-tabs` -- Tab panel with keyboard arrow navigation

**Utility helper:** Create `cn()` function in `frontend/src/lib/utils.ts` using `clsx` + `tailwind-merge` for className composition. Install `clsx` and `tailwind-merge` as dependencies.

**Variant pattern:** Use `cva` (class-variance-authority) for component variant definitions. Install `class-variance-authority` as a dependency.

## Spacing Scale

Declared values (4px base, all multiples of 4 -- D-01):

| Token | Value | CSS Variable | Usage |
|-------|-------|--------------|-------|
| xs | 4px | `--spacing-xs` | Icon gaps, inline padding |
| sm | 8px | `--spacing-sm` | Compact element spacing, icon-to-label gaps |
| xs2 | 12px | `--spacing-xs2` | Component-level horizontal padding (buttons, inputs) |
| md | 16px | `--spacing-md` | Default element padding, form field spacing |
| lg | 24px | `--spacing-lg` | Card padding, section inner padding |
| xl | 32px | `--spacing-xl` | Layout gaps between major sections |
| 2xl | 48px | `--spacing-2xl` | Page-level vertical rhythm |
| 3xl | 64px | `--spacing-3xl` | Major section breaks, sidebar width padding |

Exceptions: Touch targets for icon-only buttons use 40px minimum (10 * 4px) for accessibility compliance.

## Typography

5 scale steps (D-01 specifies 5-6 steps):

| Role | Size | Weight | Line Height | CSS Variable | Usage |
|------|------|--------|-------------|--------------|-------|
| Caption | 12px | 400 (regular) | 1.5 | `--font-caption` | Metadata, timestamps, helper text |
| Body | 14px | 400 (regular) | 1.5 | `--font-body` | Paragraph text, form labels, sidebar items |
| Subheading | 16px | 600 (semibold) | 1.4 | `--font-subheading` | Card titles, section subheadings, nav active |
| Heading | 20px | 600 (semibold) | 1.3 | `--font-heading` | Page titles, dialog headings |
| Display | 28px | 600 (semibold) | 1.2 | `--font-display` | Hero text, major feature headings |

Weights declared: 400 (regular), 600 (semibold).

Note: Existing components use `size={20}` for Lucide icons. Body text at 14px pairs well with 20px icons. Sidebar labels and result card text already use roughly these sizes.

## Shadows (D-01)

3 levels:

| Token | Value | Usage |
|-------|-------|-------|
| `--shadow-sm` | `0 1px 2px 0 rgb(0 0 0 / 0.05)` | Subtle elevation: badges, tooltips |
| `--shadow-md` | `0 4px 6px -1px rgb(0 0 0 / 0.1), 0 2px 4px -2px rgb(0 0 0 / 0.1)` | Cards, dropdowns |
| `--shadow-lg` | `0 10px 15px -3px rgb(0 0 0 / 0.1), 0 4px 6px -4px rgb(0 0 0 / 0.1)` | Dialogs, modals, floating panels |

Dark mode: Shadows use `rgb(0 0 0 / 0.3)` base for visibility against dark surfaces.

## Focal Point

**Primary screen (SearchPage):** The visual anchor is the search input bar paired with the "Suche starten" CTA button, horizontally centered in the main content area. The input occupies the majority of the horizontal space with the CTA button immediately adjacent to the right. On initial load (empty state), the search bar sits in the upper third of the content area, drawing the user's eye with the indigo-600 accent CTA against the neutral background.

### Dialog (Radix-based)

Overlay: black at 50% opacity. Content: white / dark:slate-800 background, shadow-lg, radius-lg (12px), max-width 480px, padding 24px. Close button: ghost variant, top-right corner. Animation: fade-in overlay 150ms, scale-up content 150ms (respects prefers-reduced-motion).

### Select / Dropdown (Radix-based)

Trigger styled as Input variant. Content: white / dark:slate-800, shadow-lg, radius-md. Item: 36px height, `--spacing-xs2` (12px) padding-x, hover: slate-100 / dark:slate-700. Selected item: indigo-50 background + checkmark icon.

### Tooltip (Radix-based)

Background: slate-900 / dark:slate-100. Text: white / dark:slate-900. Font size: 12px. Padding: 6px 10px. Radius: radius-sm. Delay: 400ms open, 0ms close. Arrow: same color as background. Max-width: 200px.

### Tabs (Radix-based)

List: bottom border slate-200 / dark:slate-700. Trigger: 40px height, 16px padding-x, 14px font. Inactive: slate-500 text. Active: indigo-600 text + 2px indigo-600 bottom border. Hover: slate-700 text. Content: 16px padding-top.

## Migration Contract (D-07)

### SearchPage Migration

Replace inline Tailwind utility classes with primitive components:
- Search input area -> `<Input>` primitive
- Search button -> `<Button variant="primary">` primitive
- Result items -> `<Card>` wrapping existing `ResultCard` content
- Rechtsgebiet badges -> `<Badge variant="primary">`
- Score/metadata badges -> `<Badge variant="default">`

### Sidebar Migration

Replace inline styling with token-based classes:
- Nav items use spacing tokens (sm, md) for padding
- Active state uses indigo accent color tokens
- Icon sizes remain 20px (already correct)
- Dividers use slate-200 / dark:slate-700

### Visual Regression Check

After migration, these must remain identical in behavior:
- Dark mode toggle continues working via ThemeContext
- Keyboard shortcuts (Ctrl+1-7) continue working
- Skip-link, sr-only, focus-visible, reduced-motion CSS preserved
- HealthOverlay continues rendering correctly over migrated pages
