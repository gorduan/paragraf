    **docker-compose.yml changes:**

    The current qdrant service maps `qdrant_data:/qdrant/storage` but Qdrant stores snapshots at `/qdrant/snapshots` by default, which is NOT inside `/qdrant/storage`. Per research open question #3, configure Qdrant to store snapshots inside the existing volume using an environment variable.

    Add to the qdrant service an `environment` section (it currently has none):
    ```yaml
    qdrant:
      image: qdrant/qdrant:v1.13.2
      ports:
        - "6333:6333"
      volumes:
        - qdrant_data:/qdrant/storage
      environment:
        - QDRANT__STORAGE__SNAPSHOTS_PATH=/qdrant/storage/snapshots
    ```

    This makes snapshots persist inside the existing `qdrant_data` volume, per D-01 ("already part of the qdrant_data Docker volume").
