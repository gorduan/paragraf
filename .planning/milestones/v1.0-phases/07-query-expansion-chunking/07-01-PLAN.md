3. Create `backend/src/paragraf/services/query_expander.py` with class `QueryExpander` per D-01, D-02, D-03:
   - `from __future__ import annotations` at top
   - German docstring: `"""Query Expansion – Juristisches Synonym-Woerterbuch fuer verbesserte Suche."""`
   - `__init__(self)` calls `_build_index()` then `_load_json_overrides()`
   - `_synonyms: dict[str, list[str]]` — internal synonym index (all keys lowercase)
   - `_build_index(self)`:
     - Import `LAW_REGISTRY` from `paragraf.models.law`
     - For each `abk, law_def` in `LAW_REGISTRY.items()`:
       - `self._synonyms[abk.lower()] = [law_def.beschreibung]`
       - `self._synonyms[law_def.beschreibung.lower()] = [abk]`
       - For each tag in `law_def.tags`: `self._synonyms.setdefault(tag.lower(), []).append(abk)`
     - Import `LEGAL_SYNONYMS` from `paragraf.data.synonyms`
     - For each `key, values` in `LEGAL_SYNONYMS.items()`: `self._synonyms.setdefault(key.lower(), []).extend(values)`
     - Deduplicate: for each key, `self._synonyms[key] = list(dict.fromkeys(self._synonyms[key]))` to remove duplicates while preserving order
   - `_load_json_overrides(self)`:
     - Read ENV `SYNONYMS_JSON_PATH` from settings (optional, default empty string)
     - If path is non-empty and file exists: load JSON, merge into `_synonyms` (JSON values override/extend)
     - If path is non-empty but file does NOT exist: log warning and continue (per Pitfall 6)
     - If path is empty: do nothing
   - `_normalize_query(self, query: str) -> str`:
     - Apply arabic-to-roman numeral normalization per D-03
     - Reimplement the core logic from `parser.py`'s `_normalize_abkuerzung()`: map patterns like "SGB 9" -> "SGB IX", "SGB 12" -> "SGB XII" etc.
     - Use a `ROMAN_MAP` constant: `{"1": "I", "2": "II", "3": "III", "4": "IV", "5": "V", "6": "VI", "7": "VII", "8": "VIII", "9": "IX", "10": "X", "11": "XI", "12": "XII", "14": "XIV"}`
     - Use regex `r'\b([A-Z][A-Za-z]+)\s+(\d{1,2})\b'` to find law abbreviation + arabic number patterns
     - Only replace if the result exists in LAW_REGISTRY keys (to avoid false positives)
   - `expand(self, query: str) -> tuple[str, list[str]]`:
     - First call `_normalize_query(query)` per D-03
     - Tokenize normalized query (split on whitespace)
     - For each token (lowercase), check if it exists in `_synonyms`
     - Also check multi-word combinations (2-3 consecutive tokens joined with space)
     - Collect expansion terms (max 2-3 per matched token to avoid dilution per Pitfall 1)
     - Return `(expanded_query_string, found_expansion_terms)`
     - For append strategy: expanded_query = original_normalized + " " + " ".join(found_terms)
     - This method only builds the expanded string; the caller decides strategy
   - `@property synonym_count(self) -> int`: returns `len(self._synonyms)` for diagnostics
