## Interaction Contracts

### Multi-Hop Search Flow

1. User selects "Verweissuche" in SearchModeToggle
2. DepthSlider appears below the search bar (animated slide-down, 150ms, respects prefers-reduced-motion)
3. User types query and presses Enter or clicks search button
4. Loading state: existing spinner pattern with `role="status"` and `aria-live="polite"`
5. Results render as HopResultList with hop badges color-coded by depth
6. Each result shows `via_reference` as a clickable CitationLink (if non-null)
7. Footer shows "{n} Paragraphen durchsucht" and expanded terms badges

### Depth Slider Interaction

- Range: 1 to 3 (integers only)
- Default: 1
- Label updates reactively: "Suchtiefe: 1" / "Suchtiefe: 2" / "Suchtiefe: 3"
- Accessible: `aria-label="Suchtiefe"`, `aria-valuemin="1"`, `aria-valuemax="3"`, `aria-valuenow={value}`
- Width: 120px, inline with the search controls row
- Track color: neutral-200 (inactive) / primary-500 (active fill) / dark: neutral-600 (inactive) / primary-400 (active fill)

### Post-Indexing Reference Extraction

1. Indexing queue finishes (all laws processed)
2. System adds info-level log: "Querverweise werden extrahiert..."
3. `api.extractReferences({ gesetz: null, full_reindex: false })` fires (NOT awaited in the indexing flow)
4. On success: adds info-level log with counts
5. On error: adds error-level log with message
6. No spinner or blocking — user can continue interacting with IndexPage

### State Management

- `searchMode` state extends from `"semantic" | "fulltext" | "hybrid_fulltext"` to include `"multi_hop"`
- `multiHopDepth` state: `useState<number>(1)` — local to SearchPage
- No new Context providers needed
- Multi-hop results stored in separate `useState<MultiHopResponse | null>` to avoid type collision with `SearchResponse`

## Layout Specifications

### Multi-Hop Results Layout

```
[SearchBar]
[SearchModeToggle: Semantisch | Volltext | Hybrid | Entdecken | Verweissuche]
[DepthSlider: Suchtiefe: 2  ──●──── ]          ← only visible in multi_hop mode

[HopResultCard]                                  ← full width, same max-width as ResultCard
  [HopBadge: "Direkttreffer"]  [SS 37 SGB V]
  [Titel des Paragraphen]
  [Text truncated to 3 lines...]

[HopResultCard]
  [HopBadge: "1. Verweis"]  [SS 12 SGB IX]  via SS 37 SGB V
  [Titel des Paragraphen]
  [Text truncated to 3 lines...]

[Footer: "12 Paragraphen durchsucht" | Erweitert: [GdB] [Grad der Behinderung]]
```

- HopBadge: inline-flex, rounded-full, px-2 py-1, text-caption (12px), font-semibold
- Via-reference: text-body (14px), italic, primary-600 link color, separated by 8px gap from HopBadge
- Footer: text-caption (12px), neutral-500, border-t border-neutral-200, pt-3 mt-4
- Expanded terms: inline Badge components (existing Badge primitive), neutral variant, ml-2 gap

### DepthSlider Layout

- Container: flex items-center gap-2, h-8
- Label: text-body (14px), font-regular, neutral-600 / dark: neutral-300, min-w-[100px]
- Input range: w-[120px], accent-color: primary-500
- Position: below SearchModeToggle, left-aligned with search input

## Accessibility Requirements

| Element | Requirement |
|---------|-------------|
| SearchModeToggle "Verweissuche" | `role="radio"`, `aria-checked`, keyboard navigable with arrow keys (existing pattern) |
| DepthSlider | `<input type="range">` with `aria-label="Suchtiefe"`, `aria-valuemin`, `aria-valuemax`, `aria-valuenow` |
| HopBadge | `aria-label="Hop {n}"` for screen readers, sufficient color contrast (all badge combos meet 4.5:1) |
| HopResultList | `role="list"` with `aria-label="Suchergebnisse der Verweissuche"` |
| Loading state | Existing `role="status"` + `aria-live="polite"` pattern |
| Via-reference link | Standard anchor semantics, `aria-label="Verweis auf SS X Gesetz"` |
| Extraction log entries | Rendered in existing IndexPage log with `role="log"` and `aria-live="polite"` |
