    **Root cause analysis for Gap 3 (side panel does not open on node click):**

    There are two bugs working together:

    **Bug A — GraphCanvas.tsx: Double-fire click detection race condition**

    The `handleMouseUp` (line 208-222) and `handleClick` (line 224-237) both attempt to handle clicks. When a user clicks a node:
    1. `handleMouseDown` fires, sets `dragRef.current` to the node, calls `sim.alphaTarget(0.3).restart()`
    2. `handleMouseUp` fires, checks `movedRef.current` is false, calls `onNodeClick(dragRef.current.node)`, then sets `dragRef.current = null`
    3. `handleClick` fires, checks `movedRef.current` is false, finds `dragRef.current` is null, calls `findNodeAtPoint` again and may or may not find the node (depends on whether simulation has moved the node)

    This can cause a double-call to `onNodeClick` OR miss the click entirely if the simulation moved the node between mouseDown and click.

    Fix: Remove the duplicate click logic. Use ONLY `handleMouseUp` for click detection (which already works correctly for distinguishing clicks from drags). Remove the `onClick={handleClick}` handler from the canvas entirely, or make `handleClick` a no-op when `dragRef` was involved.

    Simplest fix — change `handleClick` to only handle clicks on empty space (deselect):
    ```typescript
    const handleClick = useCallback(
      (e: React.MouseEvent<HTMLCanvasElement>) => {
        // handleMouseUp already handles node clicks via dragRef
        // This handler only fires for clicks on empty space (no node was under mouse on mouseDown)
        // Since panRef was set for empty-space mouseDown, this is already handled
      },
      [],
    );
    ```

    Or better: remove `onClick` from the canvas element entirely, since `handleMouseUp` already covers node clicks and empty-space clicks don't need special handling.

    **Bug B — GraphPage.tsx: Law-level clicks drill down instead of opening side panel**

    In GraphPage lines 396-401:
    ```typescript
    onNodeClick={(node) => {
      if (node.type === "law" && level === "laws") {
        handleDrillDown(node.id);
      } else {
        handleNodeClick(node);
      }
    }}
    ```

    At law-level, clicking a law node ALWAYS drills down — it never opens the side panel. This means at law-level, the side panel is unreachable. Per D-10, clicking a node should open a side panel.

    Fix: Make single-click open the side panel, double-click drill down. OR: at law-level, single-click opens side panel for the law (showing its reference counts), and drill-down is triggered by the "Im Detail anzeigen" button in the side panel OR by a dedicated "Paragraphen anzeigen" button.

    Simplest user-friendly fix matching D-06 and D-10: Single-click opens side panel. Add a "Paragraphen anzeigen" button in GraphSidePanel for law nodes that triggers drill-down via a new callback prop.
